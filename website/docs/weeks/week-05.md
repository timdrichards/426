# Week 05 - Compose Foundations

## Overview

Week 05 introduces Docker Compose as the baseline workflow for running and debugging multi-service systems locally.

Lecture 5.7 establishes a two-service architecture (`web` + `api`) and the core operational loop for development and troubleshooting (`up`, `ps`, `logs`, `exec`, `restart`, `down`).

The second lecture for this week is not published yet and will be added later.

## Learning Goals

By the end of this week, students should be able to:

- explain Compose terms (service, project, image, container, volume) and how they fit together
- run and inspect a multi-service system with practical Compose commands
- verify service-to-service communication through Compose networking and service-name DNS
- debug common startup and connectivity issues in a repeatable way
